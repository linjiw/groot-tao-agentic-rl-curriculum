   one missing dep.
2. **Pre-flight**: `check_environment.py --training` all-green for training
   deps under `/isaac-sim/python.sh` (Git LFS absent but unneeded here).
3. **Smoke training runs** (all `+exp/manager/universal_token/all_modes=sonic_bones_seed`,
   headless, `use_wandb=false`):
   - 64 envs × 20 iters: **~3.3 s/iter**, rewards/episode-length/adaptive-sampler
     metrics printing (`Env/adp_samp/*` — the digest's sampler stream, live).
   - 256 envs × 10 iters: **~3.7 s/iter** (Phase-2 smoke scale is comfortable).
   - GPU: fits easily in 23 GB at 256 envs.
4. **Checkpointing**: `++callbacks.model_save.save_last_frequency=5` writes
   `last.pt` (1.1 GB incl. optimizer/env state) every 5 iterations
   (`model_save_callback.py:79–87`; default cadence 50, `save_frequency` for
   step-numbered files default 2000 via `config/callbacks/model_save.yaml`).
5. **Resume/rollback**: `checkpoint=<path>/last.pt` loads and continues
   ("Loaded checkpoint from step 10") — `train_agent_trl.py:73–91`
   `resume_training` also supports auto-glob `experiment_dir_base-*/last.pt`.
   **This is the manager's rollback mechanism, verified working.**
6. **Downloads**: `sonic_release/last.pt` (448 MB) fetched from

## The launch invocation the job adapter wraps

```bash
docker exec isaac-lab-base bash -c "cd /workspace/GR00T-WholeBodyControl && \
  /isaac-sim/python.sh gear_sonic/train_agent_trl.py \
    +exp/manager/universal_token/all_modes=sonic_bones_seed \
    num_envs=256 headless=true use_wandb=false \
    project_name=<proj> experiment_name=<name> \
    base_dir=/workspace/wbc-training-logs \
    [checkpoint=<path>/last.pt] \
    ++algo.config.num_learning_iterations=<N> \
    ++algo.config.num_steps_per_env=16 \
    ++callbacks.model_save.save_last_frequency=<K>"
```

Notes for the adapter:
- Single-GPU: plain `python.sh` works; `accelerate launch` is the multi-GPU
  path — not needed at smoke scale.
- Log dir pattern: `<base_dir>/<project>/<experiment>-<YYYYmmdd_HHMMSS>/`
  (contains `config.yaml`, `meta.yaml`, `last.pt`).
- Console log carries the digest's training stream (`Mean rewards`,
  `Mean length`, `Env/adp_samp/failure_rate_*`, `Env/adp_samp/prob_*`) —
  parseable without wandb.
- Overrides trap: it's `++algo.config.num_learning_iterations`, NOT
  `trainer.max_iterations` (the 06-30 failure).
- Knob seams for the manager — **verified live this session**:
  `++manager_env.commands.motion.motion_lib_cfg.adaptive_sampling.uniform_sampling_rate=0.25`
  landed in the run's saved `config.yaml` (line 331) and trained clean
  (`wbc_knob_test` run). Same path pattern serves the cap
  (`...adaptive_sampling.adp_samp_failure_rate_max_over_mean=`) and
  `bin_size`. Family-A knobs are therefore fully drivable per-run via
  restart-with-override; within-run mutation stays future work.
