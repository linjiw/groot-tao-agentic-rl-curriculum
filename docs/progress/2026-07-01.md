# 2026-07-01 — Curriculum-Manager Agent: design (doc 08) + Phase 0 built

## What happened

1. **Design doc 08 landed** (`docs/design/08-curriculum-manager-agent.md`): the
   Curriculum-Manager Agent — an LLM supervising a *live* SONIC PPO run at
   checkpoint cadence via bounded, schema-validated deltas to the
   meta-parameters of SONIC's existing adaptive controllers. Grounded in a
   two-agent research pass:
   - Repo-side brief: verified read/write surfaces (sampler failure-rate sync,
     `im_eval_callback` metrics, `schedule_dict`, `modify_term_cfg`), TAO skill
     conventions, review constraints.
   - Literature sweep (mid-2026): Eureka→DrEureka→CurricuLLM→AURA stop at the
     per-run boundary; SONIC itself ships **no staged curriculum** (empty
     curriculum config; three always-on adaptive mechanisms). Validated
     curriculum signals for legged robots = success-band thresholds + learning
     progress; PLR/regret unproven there (LP-ACRL beat PLR) → **Stage-1 PLR
     track demoted**. Novelty: no published mid-run LLM supervision of a
     single RL run.

## Decisions

- Harness owns validation/tripwire/journal; the policy (later: LLM) is
  **never trusted** — guardrails by process boundary, not prompt.
- `allow_design=True` (design-status knobs) is replay-only.
- Eval thresholds (`terminations/tracking/eval.yaml`) are explicitly outside
  the action space (protected-metric rule).
