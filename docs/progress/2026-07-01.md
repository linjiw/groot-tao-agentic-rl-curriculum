## Decisions

- Harness owns validation/tripwire/journal; the policy (later: LLM) is
  **never trusted** — guardrails by process boundary, not prompt.
- `allow_design=True` (design-status knobs) is replay-only.
- Eval thresholds (`terminations/tracking/eval.yaml`) are explicitly outside
  the action space (protected-metric rule).

**Part 3 — infra** (`2026-07-01-infra.md`, `../infra-guide.md`): discovered

**Part 4 — `sonic-job-adapter`** (11 tests + live 3-segment validation):
KNOB_TO_HYDRA mapping, console-log→digest parser (ground-truthed on a real
excerpt), segment lifecycle with snapshots, rollback restoring pre-change
checkpoint+knobs. Two live-only bugs found+fixed (pgrep self-match;
missing-ckpt raise).

**Part 5 — ON-vs-OFF smoke** (`SMOKE_RESULTS.md`, 13 tests, 2 adversarial
review rounds → verdict COMMIT): 6 segments × 10 iters × 64 envs per arm,
seed pinned. Manager applied 2 binding-axis loosens (both `survived`);
control flat (len ~11.9) vs manager len 21.1 — **partly definitional, not
value evidence** (loosening lengthens episodes by construction). Two
design-level lessons folded into doc 08 §11 + the playbook:
1. **Binding-axis rule**: anchor_pos never fires in these runs — loosening
   it was a measured no-op; curricula must target the axis that actually
   terminates episodes (`termination_terms_mean_recent`).
2. **One-change-pending gate**: per-tick atomicity + cooldowns don't
   prevent serial multi-knob edits that orphan tripwires; the harness now
   refuses new changes while one is under watch.

**Process addition**: a project-goal-aware adversarial reviewer agent now
checks every results doc before commit (it caught the overlapping-tripwire
MAJOR that tests missed, plus 10 more findings; all addressed or queued).
